# 📘 Glossary

รายการศัพท์ที่เรียนรู้จากการฝึกงาน เขียนจากความเข้าใจและลักษณะการใช้งานจริง

---

### **API**

> Application-Programming-Interface ชื่อบริการและรูปแบบข้อมูลที่กำหนด สำหรับเขียนโปรแกรมใช้งาน มักใช้สำหรับการสื่อสารระหว่าง Backend และ Frontend หรือก็คือ เป็นตัวกลางในการส่งและเรียกข้อมูล 


### **Authentication**

> การตรวจสอบตัวตนของผู้ใช้หรือระบบ โดยตรวจสอบชื่อผู้ใช้และรหัสผ่าน

### **Frontend**

> โปรแกรมส่วนที่ทำหน้าที่แสดงผลข้อมูลให้กับผู้ใช้

### **Backend**

> โปรแกรมส่วนที่ทำหน้าที่ประมวลผล จัดเก็บข้อมูล และรับ-ส่งข้อมูลกับ Frontend

### **CLI**

> Command-Line Interface คือการสื่อสารกับระบบผ่านคำสั่งบน Terminal

### **Cache**

> ชั้นเก็บข้อมูลชั่วคราว เพื่อเพิ่มประสิทธิภาพในการเรียกใช้ข้อมูลที่ถูกใช้งานบ่อย

### **Git**

> เครื่องมือสำหรับจัดเก็บโค้ดและสำรองโค้ด

### **Fork**

> การคัดลอกสำเนาโปรเจกต์จาก GitHub มาไว้ในบัญชีของตนเอง

### **SSH**

> Secure Shell คือเครือข่ายที่ใช้ในการเชื่อมต่อและควบคุมคอมพิวเตอร์ระยะไกลอย่างปลอดภัย และ จะมีการเข้ารหัสข้อมูลที่ส่งผ่านเครือข่าย ทำให้ข้อมูลปลอดภัยจากการดักจับหรือขโมย

---

## Query SQL Command

- `SELECT` ใช้เลือกข้อมูลจาก Table
- `FROM` ใช้ระบุ Table ที่ต้องการดึงข้อมูล
- `WHERE` ใช้เลือกหรือกรองข้อมูลที่ต้องการใน Table
- `GROUP BY` ใช้จัดกลุ่มข้อมูล เพื่อไม่ให้เกิดการนับซ้ำในบางกรณี
- `ORDER BY` ใช้เรียงลำดับข้อมูล
- `LIMIT` ใช้กำหนดจำนวนข้อมูลที่ต้องการแสดง
- `UNION` ใช้รวมข้อมูลจาก 2 Table เข้าด้วยกัน โดยไม่นับข้อมูลซ้ำ
- `UNION ALL` ใช้รวมข้อมูลจาก 2 Table เข้าด้วยกัน โดยนับข้อมูลซ้ำ
- `HAVING` ใช้กรองข้อมูลคล้าย `WHERE` แต่ใช้กับข้อมูลที่ผ่านการ `GROUP BY` แล้ว
- `IF` ใช้กำหนดเงื่อนไข
- `TIMESTAMPDIFF` ใช้คำนวณความแตกต่างของเวลา
- `GROUP_CONCAT` ใช้รวมข้อมูลจาก 1 Column ขึ้นไปเข้าด้วยกัน
- `SUBSTRING_INDEX` ใช้ตัดข้อความตามจำนวนส่วนและทิศทางที่ต้องการ  
  - `1` = นับจากด้านหน้า 1 ส่วน  
  - `-1` = นับจากด้านหลัง 1 ส่วน  
  - `2` = นับจากด้านหน้า 2 ส่วน
- `CASE WHEN` ใช้กำหนดเงื่อนไข คล้าย `IF` เช่น ถ้าคะแนนมากกว่าหรือเท่ากับ 80 ให้แสดง `A` หรือถ้ามากกว่าหรือเท่ากับ 60 ให้แสดง `B`
- `XOR` ใช้คล้าย `OR` แต่จะเป็นจริงเมื่อเงื่อนไข “ต่างกัน” เท่านั้น เช่น `(a IS NULL) XOR (b IS NULL)`  
  - ✔️ `a` ว่าง แต่ `b` ไม่ว่าง  
  - ✔️ `b` ว่าง แต่ `a` ไม่ว่าง  
  - ✖️ ถ้าเหมือนกันทั้งคู่ จะไม่เป็นจริง
- `PROCEDURE` ใช้จัดเก็บ Query ที่เรียกใช้บ่อย เช่น การ Insert หรือ Update ข้อมูลรายวัน  
  วิธีใช้: `CALL procedure_name();`
- `TRUNCATE` ใช้ล้างข้อมูลทั้งหมดใน Table

## Glossary SQL

- `String` ข้อมูลประเภทข้อความ
- `Concat` คือการเชื่อมข้อความหรือข้อมูลเข้าด้วยกัน
- `Least` ใช้เลือกค่าที่น้อยที่สุด
- `Coalesce` คือการเลือกข้อมูลที่ตรงกับเงื่อนไขในวงเล็บก่อน
- `Greatest` ใช้เลือกค่าที่มากที่สุด
