# ขั้นตอนการทำงาน

## โครงการ ระบบบันทึกเวลาเข้า-ออกงาน 
git: https://github.com/koranit17k/attendance.git


## ระบบบันทึกเวลาเข้า-ออกงาน

- กำหนดหัวข้อโปรเจกต์ระบบบันทึกเวลาเข้า-ออกงาน  
- กำหนดรายละเอียดและเงื่อนไขของเวลาเข้า-ออกงาน  
- รับ Database และดำเนินการสร้าง Database รวมถึงสร้าง User สำหรับใช้งาน  
- ตรวจสอบข้อมูลและกำหนดกลุ่มของข้อมูล ได้แก่  
  - เคสที่สามารถคำนวณเวลาทำงานได้ (เคสเขียว)  
  - เคสที่ทำงานครึ่งวัน (เคสเหลือง)  
  - เคสที่ไม่สามารถคำนวณเวลาได้ (เคสแดง)  
- ตรวจสอบความถูกต้องของข้อมูล  
- Insert พนักงานที่ยังไม่มีอยู่ในตาราง `employee` และ Update ข้อมูลพนักงาน เช่น รหัสพนักงาน และ `timecode`  
- Update ข้อมูลพนักงานที่ไม่ได้มาทำงานแล้ว (ลาออก) โดยกำหนด `timecode` เป็น `0`  
- คำนวณเวลาทำงาน รวมถึงเวลามาสาย และ OT  
- จัดทำ [test case screnarios](3.exampledata.md) เพื่อระบุรูปแบบของข้อมูลที่สามารถเกิดขึ้นได้ หลังจากผ่านกระบวนการคำนวณข้อมูลแล้ว  
- ทดสอบความถูกต้องของข้อมูลด้วย SQL หรือผ่าน Vitest โดยเขียนโปรแกรมเพื่อใช้ตรวจสอบการคำนวณเวลาทำงาน เวลาสาย และ OT ของ View `vAttendance` ตาม [vitest.md](4.vitest.md)  
- ทดลองรัน Test Case โดยผลการทดสอบต้องผ่าน 100% หากใช้ Vitest ให้รันผ่านคำสั่ง `pnpm test att`  
- สร้างตาราง `attendance_clean` เพื่อใช้สำหรับปรับแก้ข้อมูล  
- จัดทำ Procedure สำหรับใช้ในการคำนวณเวลาทำงาน  
- จัดทำ Event สำหรับ Insert และ Update ข้อมูลในตาราง `attendance` โดยเรียกใช้ Procedure แบบ Auto-run เพื่อให้ระบบทำงานอัตโนมัติวันละ 1 ครั้ง และรันทุกวัน  
- จัดทำ Report  
- จัดทำระบบเรียกใช้ Report ผ่าน API โดยอ้างอิงจาก [reportworkflow.md](5.workflowreport.md)  
- จัดการระบบ localhost ของเครื่อง โดยอ้างอิงจาก [hostworkflow.md](6.workflowhost.md)  

## Note

- การสร้าง Procedure ช่วยลดระยะเวลาในการคำนวณ และช่วยลดความซับซ้อนของโค้ด  
- ในการสร้าง Event  
  - ใช้สำหรับ Insert และ Update ข้อมูลใน Database แบบอัตโนมัติผ่าน Query  
  - ครอบคลุมข้อมูล เช่น รหัสสแกน เวลา วันที่พนักงานขาดงาน รวมถึงการคำนวณเวลา  
  - ต้องระวังเรื่องการ Restart Database เพราะอาจทำให้ Event ถูกปิด (`OFF`) โดยไม่ตั้งใจ เนื่องจากระบบ Event มักไม่ค่อยถูก Support โดยอัตโนมัติในบางโปรแกรม จึงต้องตั้งค่าเพิ่มเติม  
  - ไฟล์ Config อยู่ที่ `/etc/mysql/mariadb.conf.d/50-server.cnf`  
  - ใต้บรรทัด `mysqld` ต้องเพิ่ม `event_scheduler=ON`  
  - ควรทดลอง Restart Database เพื่อยืนยันว่า `event_scheduler=ON` ยังทำงานได้ โดยใช้คำสั่ง `sudo systemctl restart mariadb`  
  - จากนั้นตรวจสอบด้วยคำสั่ง `show variables like 'event_scheduler';`  
- การใช้ Vitest ช่วยลดเวลาในการทดสอบความถูกต้องของข้อมูล และใช้งานได้สะดวกกว่าการใช้ SQL ในการรัน Test Case  
- Vitest สามารถใช้ Agent AI ใน Antigravity ช่วย Generate Test Case ที่มีอยู่แล้ว รวมถึงช่วยเขียนโค้ดได้  
- Vitest ใช้สำหรับรัน Test Case โดยผลการทดสอบต้องผ่าน 100% หากไม่ผ่าน ต้องกลับไปแก้ไข Procedure หรือ View / Query ที่ใช้คำนวณเวลา
